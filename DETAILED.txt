# 🔵 **GLOBAL RULES (APPLIES TO WHOLE SYSTEM)**

1. **NO complaint can be created unless BOTH are present:**

   * **A real photo** (mandatory)
   * **GPS coordinates** (mandatory)

   If either is missing → app blocks submission + backend returns `400`.

2. **At least ONE of the following required:**

   * Typed text description
   * Audio recording (which MUST be transcribed → translated if needed → used as description)

3. **Anonymous complaints are allowed:**

   * If no login token → create complaint with `guest_id` (string)
   * Guests **cannot upvote**, only register complaints.

4. **Clicking coordinates anywhere in the app opens Google Maps:**

   ```
   https://www.google.com/maps/search/?api=1&query=<LAT>,<LON>
   ```

5. **Duplicate detection radius:**

   * **250 meters**
   * If duplicate found → DO NOT create new complaint
   * Instead:

     * For logged-in user → auto-upvote
     * For guest → return existing complaint
   * Response includes `{ merged: true }`.

6. **Department classification logic:**

   * First try **keyword detection**
   * If NO keywords matched OR MORE THAN ONE department matches → fallback to **Gemini model**
   * Gemini returns: `{ department, problem_type }`.

7. **Audio transcription logic:**

   * If audio language = English → directly Speech-to-Text
   * If not English → Translate first → then STT → then description
   * Store BOTH raw and translated transcripts.

8. **Account privacy:**

   * Default = **private**
   * User chooses private/public ONLY during signup
   * Profile privacy CANNOT be toggled anytime later (during account creation only).

### **Audio Flow (Full)**

If audio uploaded:

1. Detect language
2. If English → STT → produce text
3. If NOT English →

   * Translate → STT → English text
4. Use resulting text as description IF user didn't type anything

### **Backend Steps After Submission**

1. Validate mandatory fields (photo + gps)
2. Handle audio transcription
3. **Department classification**:

   * Check keywords
   * If conflict/no match → Gemini
4. **Duplicate detection**:

   * Haversine distance ≤ 250m
   * Text similarity ≥ 0.3
   * If duplicate → return existing complaint
5. Upload media to Supabase storage
6. Save complaint in DB

**Result Screen:**
If duplicate → show duplicate modal
If new complaint → show success page

# 🔥 **AI ROUTING LOGIC (FULL)**

When submitting complaint:

1. Parse description (typed or transcribed)
2. Check each department keyword list
3. If EXACTLY one department matches → assign directly
4. If 0 departments OR more than 1 matched →

Prompt to Gemini:
“Given this description: <text>, choose only one department from this list: [...] and provide problem_type as short category.”

5. Use Gemini result
6. If Gemini fails → fallback to Sanitation

---

# 🔥 **DUPLICATE DETECTION — FULL LOGIC**

Backend does:

1. Get GPS
2. Fetch complaints within ~300m bounding box
3. Calculate haversine distance
4. If ≤ 250m → compute text similarity
5. If similarity ≥ 0.3 → duplicate
6. Behavior:

   * If citizen logged in → auto-upvote existing complaint
   * If guest → return existing complaint

Return:

```
{ merged: true, existing_complaint: {...} }
```
